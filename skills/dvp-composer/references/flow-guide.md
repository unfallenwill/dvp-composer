# 五阶段交互流程指南

DVP Composer 的完整交互流程。每个阶段结束时执行对应的质量门禁。

---

## 阶段一：文档收集与信息提取

### 目标

收集并提取 Protocol、CRF、DMP 等参考文档中的关键信息，为后续阶段提供预填充基础。

### 步骤

1. **询问参考文档**：
   - "是否有 Protocol（研究方案）文件？请提供路径。"
   - "是否有 CRF（病例报告表）文件？请提供路径。"
   - "是否有 DMP（数据管理计划）文件？请提供路径。"
   - "如果都没有，可以直接进入交互式问答模式。"

2. **读取和提取**：
   - 用 Read 工具读取文档（支持 PDF、Word、Excel、Markdown 等）
   - 对于 Excel 格式的 CRF，用 `python3 -c "..."` 提取结构信息
   - 提取的信息按类别组织：
     - **Protocol 信息**：研究名称、编号、设计类型、阶段、终点、访视计划、入排标准
     - **CRF 信息**：表单名称、字段列表、字段类型、选项值
     - **DMP 信息**：数据管理流程、质量标准、时间节点

3. **存储提取结果**：
   - 将提取的信息写入临时 JSON 文件（`/tmp/dvp_extracted.json`），供后续阶段使用

### 门禁 G1：信息完整性确认

- 分类汇总展示所有提取的信息
- 每条信息标注来源（Protocol / CRF / DMP）
- 标注缺失的关键字段（用红色标记或明确列出）
- 向用户展示：
  ```
  === 信息提取汇总 ===

  来源: Protocol
  ✅ 研究名称: ABC-123
  ✅ 方案编号: PROTO-2024-001
  ✅ 研究阶段: III期
  ⚠️ 缺失: 目标入组人数
  ⚠️ 缺失: 研究中心列表

  来源: CRF
  ✅ 表单数量: 12
  ✅ 字段总数: 156
  ⚠️ 缺失: 部分字段的选项值未提取到
  ```
- 用户确认或补充后，进入阶段二

### 错误恢复

- 文件无法读取：提示用户检查路径和格式，或跳过该文档
- 提取信息不完整：在缺失处标注，可在后续阶段补充

---

## 阶段二：结构发现

### 目标

确定 DVP 的章节结构和内容组织方式。

### 步骤

1. **提供三种来源选项**：

   **选项 A — 现有模板**：
   - "请提供现有的 DVP 模板文件路径"
   - 读取模板，提取章节结构
   - 展示提取结果供确认

   **选项 B — 从常见章节选择**：
   - 读取 [section-catalog](section-catalog.md)
   - 展示章节目录，标注哪些有文档信息可以预填充
   - 用户勾选需要的章节
   - 示例展示：
     ```
     === 可选章节 ===
     (P=有Protocol信息, C=有CRF信息, D=有DMP信息)

     [P] 1. 研究信息
     [P] 2. 数据验证范围
     [P] 3. 访视计划
     [C] 4. 编辑检查
     [D] 5. 数据录入指南
     [D] 6. Query 管理
     [ ] 7. SDTM 映射
     [ ] 8. 签字与审批

     请选择要包含的章节（输入编号，或描述自定义章节）：
     ```

   **选项 C — 用户口述**：
   - "请描述你需要的 DVP 章节结构"
   - 解析用户描述，生成章节列表
   - 与 section-catalog 匹配已有章节类型

2. **确认章节列表**：
   - 展示最终章节列表（编号、名称、类型）
   - 允许调整顺序、增删、改名

### 门禁 G2：结构确认

- 展示完整章节表格
- 要求用户**显式确认**（回复"确认"、"OK"、"可以"、"没问题"等肯定表述均可）
- 未确认不允许进入阶段三

---

## 阶段三：逐章节收集数据

### 目标

为每个确认的章节填充具体内容。

### 通用原则

- **一次一个主题**，不一次性展示多个章节的表单
- **预填充优先**：先展示文档提取的已有信息，用户确认或修改
- **支持跳过**：用户可说"跳过"或"稍后填写"，该章节留占位符
- **支持批量粘贴**：表格类章节允许用户直接粘贴多行数据

### 各类型的填充策略

#### key-value 章节

1. 展示已提取的字段值（如有）
2. 逐项或分批确认
3. 缺失字段逐个询问
4. 示例对话：
   ```
   === 章节 1/8: 研究信息 ===

   以下信息已从 Protocol 中提取：
   - 研究名称: ABC-123 ✅
   - 方案编号: PROTO-2024-001 ✅
   - 研究阶段: III期 ✅
   - 研究设计: (未提取到)

   请确认以上信息，并补充缺失项。
   研究设计是什么？（如：随机、双盲、安慰剂对照）
   ```

#### table 章节

1. 先确认列头（是否需要调整）
2. 展示已提取的数据行
3. 询问是否有补充或修改
4. 支持批量粘贴格式说明：
   ```
   请提供编辑检查数据。可以用以下格式粘贴多行：
   检查ID | 变量 | 条件 | 消息 | 严重程度
   EC001 | DOB | > 研究日期 | 出生日期晚于研究日期 | 错误
   ```

#### narrative 章节

1. 如果有文档信息，生成草稿供用户确认
2. 如果没有，通过开放式问答引导用户描述
3. 示例对话：
   ```
   === 章节: 背景 ===

   请描述本研究的背景信息。可包含：
   - 研究目的
   - 数据验证的目标
   - 适用范围
   ```

### 门禁 G3：内容完整性检查

每个章节填充后做即时检查：
- 必填项是否有值（必填项定义见 section-catalog.md，用户可调整）
- 缺失项标红提醒

全部章节完成后，展示汇总视图：
```
=== 内容填充汇总 ===

1. 研究信息      ✅ 完成 (8/8 字段)
2. 数据验证范围   ✅ 完成
3. 访视计划       ✅ 完成 (6 行数据)
4. 编辑检查       ⚠️ 部分填充 (12/15 行, 3 行缺失严重程度)
5. 数据录入指南   ⏭️ 跳过
6. Query 管理     ✅ 完成

请确认后进入逻辑审查阶段。是否需要返回修改？
```

用户确认后进入阶段四。

---

## 阶段四：逻辑审查

### 目标

系统性校验 DVP 中数据的逻辑一致性。

### 步骤

1. **读取审查规则**：[review-rules](review-rules.md)
2. **逐项执行检查**（R1-R6）
3. **生成审查报告**

### 审查报告输出格式

```
=== DVP 逻辑审查报告 ===

R1 字段引用完整性
  ✅ [具体结果]
  ❌ [具体问题描述]

R2 数据一致性
  ✅ [具体结果]
  ⚠️ [具体问题描述]

...（R3-R6 同理）

---
总计：✅ X  ⚠️ Y  ❌ Z
```

### 门禁 G4：审查通过

- 所有 ❌ 错误项**必须修正**
- ⚠️ 警告项由用户决定：修正或标记为"已知并接受"
- 修正后可重新执行审查
- 全部处理后用户确认，进入阶段五

---

## 阶段五：生成输出

### 目标

将已验证的 DVP 内容渲染为格式化的 Excel 文件。

### 步骤

1. **组装 JSON**：将所有收集的数据组装为 JSON 格式（参考 [excel-spec](excel-spec.md)）
2. **确认输出路径**：
   - 默认：`./DVP_<研究名>_<日期>.xlsx`
   - 用户可自定义路径
3. **写入临时 JSON 文件**
4. **执行生成脚本**：
   ```
   python3 ${CLAUDE_SKILL_DIR}/scripts/generate_xlsx.py /tmp/dvp_data.json --output <输出路径>
   ```
5. **验证生成结果**

### 门禁 G5：输出验证

- 确认 .xlsx 文件已成功生成
- 展示预览摘要：
  ```
  === 生成完成 ===
  文件: ./DVP_ABC-123_2026-04-16.xlsx
  章节: 8 个
  总行数: 45 行
  Sheet: DVP (单 sheet)

  请打开文件确认格式和内容。需要调整吗？
  ```
- 用户确认满意或返回修改

### 后续选项

- "调整格式" — 修改颜色、字体等
- "补充章节" — 返回阶段三添加新章节
- "修正内容" — 返回阶段三修改特定章节
- "重新审查" — 返回阶段四重新执行逻辑审查
- "完成" — 结束 Skill 执行
