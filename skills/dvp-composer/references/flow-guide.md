# 流程指南

DVP Composer 的工作指南。按信息收集 → 审查 → 输出的顺序组织，描述各环节的具体做法。

---

## 文档收集与信息提取

### 做法

1. 如果用户提供了参考文档（Protocol / CRF / DMP）：
   - 用 Read 读取文档
   - 对于 Excel 格式的 CRF，用 `python3 -c "..."` 提取结构信息
   - 提取的信息按类别组织：
     - **Protocol 信息**：研究名称、编号、设计类型、阶段、终点、访视计划、入排标准
     - **CRF 信息**：表单名称、字段列表、字段类型、选项值
     - **DMP 信息**：数据管理流程、质量标准、时间节点

2. 将提取的信息写入 `/tmp/dvp_extracted.json`

3. 如果没有提供参考文档，直接进入章节结构确定环节

### 提取结果呈现

分类汇总展示所有提取的信息，标注来源和缺失项：

```
=== 信息提取汇总 ===

来源: Protocol
✅ 研究名称: ABC-123
✅ 方案编号: PROTO-2024-001
✅ 研究阶段: III期
⚠️ 缺失: 目标入组人数
⚠️ 缺失: 研究中心列表

来源: CRF
✅ 表单数量: 12
✅ 字段总数: 156
⚠️ 缺失: 部分字段的选项值未提取到
```

如果缺失项较多，询问用户是否补充。如果信息已足够，直接继续。

### 异常处理

- 文件无法读取：提示用户检查路径和格式，或跳过该文档
- 提取信息不完整：在缺失处标注，可在后续环节补充

## 章节结构确定

### 做法

根据用户意图确定 DVP 章节结构：

**用户提供了现有模板**：
- 读取模板，提取章节结构
- 展示提取结果供确认

**用户未指定结构**：
- 读取 [section-catalog](section-catalog.md)
- 展示章节目录，标注哪些有文档信息可以预填充

**用户直接描述结构**：
- 解析用户描述，与 section-catalog 匹配章节类型

展示示例：
```
=== 可选章节 ===
(P=有Protocol信息, C=有CRF信息, D=有DMP信息)

[P] 1. 研究信息
[P] 2. 数据验证范围
[P] 3. 访视计划
[C] 4. 编辑检查
[D] 5. 数据录入指南
[D] 6. Query 管理
[ ] 7. SDTM 映射
[ ] 8. 签字与审批
```

用户确认后列出最终章节（编号、名称、类型），允许调整。

## 章节内容填充

### 通用原则

- **一次一个主题**，不一次性展示多个章节的表单
- **预填充优先**：先展示文档提取的已有信息，用户确认或修改
- **支持跳过**：用户可说"跳过"或"稍后填写"，该章节留占位符
- **支持批量粘贴**：表格类章节允许用户直接粘贴多行数据

### 各类型的填充方式

#### key-value 章节

1. 展示已提取的字段值（如有）
2. 逐项或分批确认
3. 缺失字段逐个询问
4. 示例：
   ```
   === 章节 1/8: 研究信息 ===

   以下信息已从 Protocol 中提取：
   - 研究名称: ABC-123 ✅
   - 方案编号: PROTO-2024-001 ✅
   - 研究阶段: III期 ✅
   - 研究设计: (未提取到)

   请确认以上信息，并补充缺失项。
   研究设计是什么？（如：随机、双盲、安慰剂对照）
   ```

#### table 章节

1. 先确认列头（是否需要调整）
2. 展示已提取的数据行
3. 询问是否有补充或修改
4. 支持批量粘贴格式说明：
   ```
   请提供编辑检查数据。可以用以下格式粘贴多行：
   检查ID | 变量 | 条件 | 消息 | 严重程度
   EC001 | DOB | > 研究日期 | 出生日期晚于研究日期 | 错误
   ```

#### narrative 章节

1. 如果有文档信息，生成草稿供用户确认
2. 如果没有，通过开放式问答引导用户描述
3. 示例：
   ```
   === 章节: 背景 ===

   请描述本研究的背景信息。可包含：
   - 研究目的
   - 数据验证的目标
   - 适用范围
   ```

### 完成后汇总

全部章节完成后展示汇总视图：
```
=== 内容填充汇总 ===

1. 研究信息      ✅ 完成 (8/8 字段)
2. 数据验证范围   ✅ 完成
3. 访视计划       ✅ 完成 (6 行数据)
4. 编辑检查       ⚠️ 部分填充 (12/15 行, 3 行缺失严重程度)
5. 数据录入指南   ⏭️ 跳过
6. Query 管理     ✅ 完成
```

如有缺失或跳过，询问用户是否需要补充。

## 逻辑审查

### 做法

1. 读取审查规则：[review-rules](review-rules.md)
2. 逐项执行检查（R1-R6）
3. 生成审查报告

### 审查报告格式

```
=== DVP 逻辑审查报告 ===

R1 字段引用完整性
  ✅ [具体结果]
  ❌ [具体问题描述]

R2 数据一致性
  ✅ [具体结果]
  ⚠️ [具体问题描述]

...（R3-R6 同理）

---
总计：✅ X  ⚠️ Y  ❌ Z
```

- ❌ 错误项必须修正
- ⚠️ 警告项由用户决定：修正或标记为"已知并接受"
- 修正后可重新执行审查

---

## 生成输出

### 做法

1. 组装 JSON 数据（参考 [excel-spec](excel-spec.md)）
2. 确认输出路径（默认 `./DVP_<研究名>_<日期>.xlsx`）
3. 写入临时 JSON 文件
4. 执行生成脚本：
   ```
   python3 ${CLAUDE_SKILL_DIR}/scripts/generate_xlsx.py /tmp/dvp_data.json --output <输出路径>
   ```
5. 验证生成结果

### 生成结果呈现

```
=== 生成完成 ===
文件: ./DVP_ABC-123_2026-04-16.xlsx
章节: 8 个
总行数: 45 行
Sheet: DVP (单 sheet)
```

### 后续选项

- "调整格式" — 修改颜色、字体等
- "补充章节" — 添加新章节
- "修正内容" — 修改特定章节
- "重新审查" — 重新执行逻辑审查
- "完成" — 结束
