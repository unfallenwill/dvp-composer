# 阶段 2：探索

## 职责

分析收集到的资料，理解研究全貌，发现文档间的冲突和歧义，评估信息完备性。

## 输入

- `/tmp/dvp_raw_materials.json`（阶段 1 的输出）
- 用户在阶段 1 中提供的所有文档和信息

## 做法

### 1. 理解资料

梳理收集到的所有信息，形成对研究的整体理解：

- 研究设计和复杂度（如 III 期、多中心 → 意味着数据量大、中心多）
- 涉及的数据域范围（从 CRF 表单推断）
- 关键验证对象（从 Protocol 的终点和入排标准推断）
- 数据管理要求（从 DMP 推断）

### 2. 发现冲突

检查不同来源之间的不一致：

- Protocol 与 CRF 之间：如 Protocol 描述的访视数与 CRF 表单不匹配
- Protocol 与 DMP 之间：如 Protocol 要求的质量标准与 DMP 定义不同
- CRF 与 DMP 之间：如 CRF 字段定义与 DMP 的数据流程矛盾
- 用户补充信息与文档之间：如用户说的与文档记载不符

发现的冲突必须向用户展示并请求澄清。

### 3. 发现歧义

检查同一信息的多种可能解读：

- 模糊的描述（如"适时评估"→ 具体什么时候？）
- 不明确的业务规则（如"严重不良事件需报告"→ 报告给谁、时限多久？）
- 字段语义不确定（如某个 CRF 字段的含义可能有多种理解）

歧义项向用户提问澄清，每次一个主题。用户可以说"稍后处理"跳过。

### 4. 评估完备性

按数据域评估信息的充分程度：

- 信息充足的数据域 → 标注为"可生成"
- 信息部分缺失的数据域 → 标注为"部分可生成"，列出缺失项
- 信息完全缺失的数据域 → 标注为"需用户补充"

## 输出

更新 `/tmp/dvp_raw_materials.json`，追加探索结果：

```json
{
  "exploration": {
    "study_overview": "研究整体概貌描述",
    "complexity": "高/中/低",
    "key_domains": ["人口学", "生命体征", "AE", ...],
    "conflicts": [
      {"source_a": "Protocol", "source_b": "CRF", "description": "冲突描述", "resolution": "用户回答或待定"}
    ],
    "ambiguities": [
      {"topic": "歧义主题", "question": "澄清问题", "resolution": "用户回答或跳过"}
    ],
    "domain_readiness": [
      {"domain": "人口学", "status": "可生成"},
      {"domain": "AE", "status": "部分可生成", "missing": ["因果关系评估规则"]}
    ]
  }
}
```

## 展示格式

```
=== 探索结果 ===

研究概貌: ABC-123，III期，随机双盲安慰剂对照多中心
复杂度: 高（多中心、多访视、多数据域）

⚠️ 冲突 (2):
  1. Protocol 描述 7 个访视，CRF 仅有 6 个表单 → 缺少 Follow-up 表单？
  2. DMP 要求 Query 5 个工作日回复，用户说 7 个 → 以哪个为准？

⚠️ 歧义 (1):
  1. "适时评估疗效" → 具体是哪些访视需要评估？

数据域完备性:
  ✅ 人口学、生命体征、AE → 可生成
  ⚠️ 实验室检查 → 部分可生成（缺少正常范围定义）
  ❌ 合并用药 → 需用户补充

请处理冲突和歧义，或直接进入模板确认。
```

## 完成条件

- 冲突项已展示给用户，用户已回应或标记为待定
- 歧义项已提问，用户已回答或跳过
- 研究概貌和数据域完备性评估完成

## 回退

用户补充新资料时，回到阶段 1。在阶段 1 中更新资料后重新进入本阶段。
