# 阶段 1：资料收集

## 目标

从用户提供的研究材料中，提取出编写 DVP 所需的完整信息。

## 流程

1. 向用户索要 Protocol、CRF、Edit Check Spec 等常见材料。告诉用户：缺的不要紧，先给有的。
2. 通读用户提供的每一份文件，理解研究设计、数据结构和验证需求。
3. 对照已读内容，定位具体缺失——指明"哪个模块缺什么"，而非泛泛追问。
4. 每条追问都指向明确的文件名或内容区域，让用户一看就知道该找什么。
5. 重复步骤 2–4，直到信息足以支撑 DVP 生成。

## 完成条件

- 至少掌握足够确定 DVP 模板结构的信息
- 无法获取的信息已标注为缺失，但不阻塞流程

## 输出

写入 `/tmp/dvp_raw_materials.json`：

```json
{
  "sources": {
    "protocol": { "extracted": {...}, "missing": [...] },
    "crf": { "extracted": {...}, "missing": [...] },
    "dmp": { "extracted": {...}, "missing": [...] }
  },
  "user_supplied": {
    "field1": "value1"
  },
  "clarifications": [
    { "question": "...", "answer": "..." }
  ]
}
```

## 异常处理

- 文件无法读取：提示用户检查路径和格式，或跳过该文档
- 提取信息不完整：在缺失处标注，可在后续环节补充
