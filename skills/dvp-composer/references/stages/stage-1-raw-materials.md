# 阶段 1：资料收集

## 职责

收集生成 DVP 所需的所有原始材料。

## 输入

- 用户提供的参考文档（Protocol / CRF / DMP，可选任意组合）
- 用户口头补充的信息
- 用户对歧义的澄清回答

## 做法

### 1. 读取并提取文档信息

如果用户提供了参考文档：

- **Protocol**：用 Read 读取，提取研究名称、编号、设计类型、阶段、终点、访视计划、入排标准
- **CRF**：用 Read 读取；Excel 格式的 CRF 用 `python3` 提取结构信息（表单名称、字段列表、字段类型、选项值）
- **DMP**：用 Read 读取，提取数据管理流程、质量标准、时间节点

如果用户未提供文档，跳过此步骤，直接进入信息确认。

### 2. 整理用户口头补充的信息

用户在对话中提到的任何信息（如"这个研究是 III 期"、"用 MedDRA 编码"）直接纳入。

### 3. 澄清歧义

信息不完整或有歧义时向用户提问，每次只澄清一个主题。用户可随时补充更多信息。

### 4. 判断是否足够

判断当前信息是否足够推进到模板确认：
- **够**：至少掌握研究基本信息 + 至少一个数据源的领域信息
- **不够**：继续收集或向用户说明缺失项

采用乐观推进策略——能走就走，不够就记。不要求所有信息完备才进入下一阶段。

## 输出

写入 `/tmp/dvp_raw_materials.json`：

```json
{
  "sources": {
    "protocol": { "extracted": {...}, "missing": [...] },
    "crf": { "extracted": {...}, "missing": [...] },
    "dmp": { "extracted": {...}, "missing": [...] }
  },
  "user_supplied": {
    "field1": "value1"
  },
  "clarifications": [
    { "question": "...", "answer": "..." }
  ]
}
```

## 完成条件

- 至少掌握足够确定 DVP 模板结构的信息
- 无法获取的信息已标注为缺失，但不阻塞流程

## 展示格式

```
=== 资料收集汇总 ===

来源: Protocol
  ✅ 研究名称: ABC-123
  ✅ 方案编号: PROTO-2024-001
  ⚠️ 缺失: 目标入组人数

来源: CRF
  ✅ 表单数量: 12
  ✅ 字段总数: 156

来源: 用户补充
  ✅ 研究阶段: III期

⚠️ 缺失项: 目标入组人数, 研究中心列表
是否继续？补充缺失项或直接进入模板确认。
```

## 异常处理

- 文件无法读取：提示用户检查路径和格式，或跳过该文档
- 提取信息不完整：在缺失处标注，可在后续环节补充
