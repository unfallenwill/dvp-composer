# DVP 常见章节目录

本目录是**建议性参考**，不是固定模板。DVP 结构由用户在交互过程中确定。

每章节标注**数据来源**：Protocol（P）、CRF（C）、DMP（D）、手动输入（M）。

---

## 1. 研究信息（Study Information）

**类型**：key-value

**典型字段**：

| 字段 | 数据来源 | 说明 |
|------|---------|------|
| 研究名称 | P | 研究标题全称 |
| 方案编号 | P | 唯一标识 |
| 研究阶段 | P | I/II/III/IV 期 |
| 研究设计 | P | 随机、双盲、安慰剂对照等 |
| 适应症 | P | 目标疾病/症状 |
| 目标入组人数 | P | 计划入组受试者数 |
| 研究中心数 | P | 参与研究的研究中心数量 |
| 申办方 | P | Sponsor 名称 |
| CRO | P/M | 负责 CRO 名称 |
| 数据经理 | M | DM 姓名及联系方式 |

**建议问题**：
- "请提供研究名称和方案编号"
- "本研究是什么阶段、什么设计类型？"

## 2. 数据验证范围（Scope of Data Verification）

**类型**：narrative + key-value

**典型字段**：

| 字段 | 数据来源 | 说明 |
|------|---------|------|
| 验证范围 | P/D | 包含哪些数据域/模块 |
| EDC 系统 | D | 使用的电子数据采集系统 |
| 数据库版本 | D | 临床数据库版本号 |
| 验证起始日期 | M | DVP 执行开始日期 |
| 验证截止日期 | M | DVP 执行截止日期 |

**建议问题**：
- "本次 DVP 覆盖哪些数据域？"
- "使用什么 EDC 系统？"

---

## 3. 访视计划（Visit Schedule）

**类型**：table

**典型字段**：

| 字段 | 数据来源 | 说明 |
|------|---------|------|
| 访视名称 | P | 如 Screening、Visit 1、Visit 2 |
| 访视窗口 | P | 相对时间窗口（如 ±3 天） |
| 访视天数 | P | 相对于基线的天数 |
| 数据域 | P/C | 该访视需要收集的数据域 |

**建议问题**：
- "研究包含哪些访视？"
- "每个访视的时间窗口是什么？"

---

## 4. 编辑检查（Edit Checks）

**类型**：table

**典型字段**：

| 字段 | 数据来源 | 说明 |
|------|---------|------|
| 检查 ID | M | 唯一标识（如 EC001） |
| 数据域 | C | 所属表单/域 |
| 变量名 | C | 被检查的变量 |
| 检查条件 | M/P/C | 逻辑条件表达式 |
| 错误消息 | M | 触发时的提示消息 |
| 严重程度 | M | 错误/警告/信息 |
| 检查类型 | M | 范围检查/逻辑检查/一致性检查 |

**建议问题**：
- "需要定义哪些编辑检查？"
- "每个检查的逻辑条件和严重程度是什么？"
- "是否要批量粘贴检查列表？"
